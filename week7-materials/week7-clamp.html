1. The clamp() Function

‚úì With clamp() - Fluid Scaling

This Title Uses clamp()

This text scales smoothly between minimum and maximum values. It never gets too small or too large, ensuring readability across all screen sizes.

/* min, preferred, max */ font-size: clamp(1rem, 2.5vw, 2rem);

‚úó Fixed Size - No Scaling

This Title is Fixed

This text stays the same size regardless of screen width. On mobile, it might be too large. On large screens, it might look tiny.

/* Always 24px - never changes */ font-size: 24px;

How clamp() Works:

font-size: clamp(MIN, PREFERRED, MAX);
  • MIN (1rem): Smallest size the text can be (prevents text from being unreadable)
  • PREFERRED (2.5vw): The ideal size that scales with viewport (fluid)
  • MAX (2rem): Largest size the text can be (prevents text from being huge)

2. Viewport Units (vw, vh, vmin, vmax)

vw (Viewport Width)

3vw Text
font-size: 3vw;

3% of viewport width. Gets bigger when you make the window wider. Changes with: Window width only

vh (Viewport Height)

3vh Text
font-size: 3vh;

3% of viewport height. Gets bigger when you make the window taller. Changes with: Window height only

vmin (Viewport Minimum)

3vmin Text
font-size: 3vmin;

3% of the smaller dimension (width or height). Use for: Elements that should scale proportionally

vmax (Viewport Maximum)

3vmax Text
font-size: 3vmax;

3% of the larger dimension (width or height). Use for: Hero text, splash screens

‚ö†Ô∏è Viewport Units Alone Can Be Problematic:

  • Too small on mobile: 2vw on a 320px phone = 6.4px (unreadable!)
  • Too large on desktop: 2vw on a 2560px monitor = 51.2px (huge!)
  • Solution: Use clamp() to set boundaries!

3. Best Practices for Responsive Typography

‚úì Recommended Approaches:

/* Headings - scales between 24px and 48px */ h1 { font-size: clamp(1.5rem, 4vw, 3rem); } /* Body text - scales between 16px and 20px */ body { font-size: clamp(1rem, 1.5vw, 1.25rem); } /* Small text - scales between 14px and 16px */ .small { font-size: clamp(0.875rem, 1.2vw, 1rem); }

‚úì DO: Use clamp() for Fluid Typography

  • Scales smoothly across all screens
  • Maintains readability
  • Reduces need for media queries
  • Better user experience

‚úó DON'T: Use Pure Viewport Units

  • Can become unreadable on small screens
  • Can be too large on big screens
  • Accessibility issues
  • Fails WCAG guidelines

üéØ Real-World Use Cases:

  • Hero headings: clamp(2rem, 5vw, 4rem)
  • Section titles: clamp(1.5rem, 3vw, 2.5rem)
  • Body text: clamp(1rem, 1.5vw, 1.125rem)
  • Card content: clamp(0.875rem, 1.2vw, 1rem)
